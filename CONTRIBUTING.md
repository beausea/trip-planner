# 기여 가이드 (Contributing Guide)

가족 여행 플래너 AI에 기여해 주셔서 감사합니다!
이 프로젝트는 **한국의 부모들이 함께 만들어 가는 여행 교육 자료 모음**입니다.

---

## 누구나 기여할 수 있습니다

- 직접 다녀온 여행지의 교육 자료를 만든 분
- 아이와 함께 풀었던 퀴즈를 공유하고 싶은 분
- 더 좋은 여행 경로나 맛집 정보가 있는 분
- 시스템 프롬프트를 개선하고 싶은 개발자

---

## 기여 방법 1: 샘플 여행 계획서 추가

가장 환영하는 기여입니다. **직접 다녀온 여행의 계획서**를 공유해 주세요.

### 작성 방법

1. `opensource/samples/` 폴더에 새 파일을 만드세요.
2. 파일명 규칙: `{지역}-{일수}.md`

   ```
   예: jeonju-1day.md        (전주 당일치기)
       jeju-3nights-4days.md  (제주 3박4일)
       andong-1night-2days.md (안동 1박2일)
   ```

3. 아래 **필수 섹션**을 포함해야 합니다:

   ```markdown
   ## 여행 개요
   (기간, 인원, 테마, 예상 비용)

   ## Day N 시간표
   (이동 경로, 시간표)

   ## Day N 교육 자료
   (여행지별 — 이야기, 관람 포인트, 아이에게 물어볼 질문)

   ## Day N 저녁 퀴즈
   (5문제 이상, 정답+해설 포함)
   ```

4. `samples/gyeongju-2nights-3days.md`를 참고 템플릿으로 사용하세요.

### 교육 자료 작성 원칙

| 항목 | 기준 |
|------|------|
| 대상 | 초등 5학년 기준 (초등 3~6학년 폭넓게) |
| 언어 | 쉽고 친근한 한국어 |
| 비율 | 아이 눈높이 80% + 성인 심화 20% |
| 형식 | 이야기체 (딱딱한 나열 금지) |
| 질문 | "왜?", "만약에?" 형식으로 호기심 유발 |
| 출처 | 공식 사이트 URL 반드시 포함 |

### 피해야 할 것

- ❌ 나무위키·위키피디아를 유일한 출처로 사용하는 것
- ❌ 확인되지 않은 운영 시간·입장료 단정
- ❌ 아이가 이해하기 어려운 전문 용어 남발
- ❌ 교육 자료 없이 일정표만 작성

## 기여 방법 2: 기존 내용 개선

이미 있는 샘플에서 아래 항목을 개선할 수 있습니다:

- 운영 시간·입장료 최신화
- 더 재미있는 퀴즈 문제 추가
- 교육 자료 내용 보강
- 오탈자·사실 오류 수정

GitHub Issues에 "수정 제안: {파일명}" 형식으로 제보해 주세요.

---

## 기여 방법 3: 시스템 프롬프트 개선

`tier1/` 폴더의 프롬프트를 개선하는 기여도 환영합니다.

### 제안 방법

1. GitHub Issues에 개선 제안 작성
2. 제안 형식:
   ```
   [개선 제안] {제목}

   현재 동작:
   (문제가 되는 현재 프롬프트 내용이나 AI 응답 예시)

   제안 내용:
   (어떻게 바꾸면 좋은지)

   기대 효과:
   (개선 후 어떤 점이 나아지는지)
   ```

### 개선이 필요한 알려진 한계

- [ ] AI가 실제 운영 시간을 잘못 알 수 있음 → 항상 "재확인 필요" 안내 강화
- [ ] 일부 지역의 교육 자료가 부족함 → 샘플 추가로 해결 가능
- [ ] 예산 계산이 부정확할 수 있음 → 범위(최소~최대)로 표시 권장

## 기여 방법 4: 지역별 공식 소스 목록 추가

각 지역의 공식 관광 사이트를 알고 계시면 알려주세요.

현재 사용 중인 공식 소스:

| 기관 | URL | 활용 범위 |
|------|-----|----------|
| 한국관광공사 | korean.visitkorea.or.kr | 전국 여행지 정보 |
| 국가유산청 | heritage.go.kr | 문화재·유적 정보 |
| 국립중앙박물관 | museum.go.kr | 박물관 전시 정보 |
| 경주시 관광 | guide.gyeongju.go.kr | 경주 전용 |
| 전주시 관광 | tour.jeonju.go.kr | 전주 전용 |

추가 지역 사이트는 Issue로 제보해 주세요.

## Pull Request 절차

1. 이 저장소를 Fork합니다
2. 새 브랜치 생성: `git checkout -b add-{지역명}-sample`
3. 파일 작성 후 커밋: `git commit -m "add: {지역} {일수} 샘플 계획서"`
4. PR 제목 형식: `[샘플 추가] {지역} {일수} — {테마}`
5. PR 본문에 아래를 포함해 주세요:
   - 실제 방문 여부 (직접 다녀온 여행이면 아이 반응도)
   - 정보 출처 목록
   - 특별히 강조하고 싶은 교육 포인트

---

## 행동 강령

- **정확성**: 확인되지 않은 정보는 추측이라고 명시하거나 포함하지 마세요
- **존중**: 모든 가족 형태와 여행 스타일을 존중해 주세요
- **안전**: 아이 동반 여행에 안전하지 않은 정보는 포함하지 마세요
- **한국어**: 이 프로젝트는 한국어를 기본으로 합니다

## 자주 묻는 질문

**Q: 사진을 포함해도 되나요?**
저작권 문제로 직접 찍은 사진만 포함 가능합니다. 현재는 텍스트 위주로 유지합니다.

**Q: 해외 여행지 샘플도 추가할 수 있나요?**
현재는 국내 여행에 집중하고 있습니다. 향후 별도 폴더로 관리할 예정입니다.

**Q: 유치원생 자녀에게도 맞나요?**
현재 샘플은 초등 5학년 기준이지만, 더 어린 연령대용 버전을 작성해서 기여하실 수 있습니다.
파일명에 `{지역}-{일수}-kindergarten.md` 처럼 대상 연령을 표시해 주세요.
